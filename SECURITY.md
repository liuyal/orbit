# Security Policy

## Supported Versions

We are currently only patching security vulnerabilities in the latest major release of the project.

| Version | Supported |
| :--- | :--- |
| 2.x | Yes |
| 1.x | No |

## Reporting a Vulnerability

If you discover a security vulnerability within this repository, please report it to us as soon as possible. We take all security reports seriously and appreciate your efforts in helping us maintain the security of our project.

### How to Report

1.  **Do not** open a public GitHub issue or discuss the vulnerability in public forums.
2.  Report the vulnerability via our private reporting channel: [Replace this with your preferred reporting method, e.g., security@example.com or a link to a private vulnerability reporting form/page].
3.  In your report, please include:
    *   A clear and concise description of the vulnerability.
    *   Steps to reproduce the vulnerability, including any necessary proof-of-concept code or scenarios.
    *   The potential impact of the vulnerability.
    *   Any suggested mitigations (if known).

### Our Commitment

*   We will acknowledge receipt of your report within [e.g., 2 business days].
*   We will provide an estimated timeline for addressing the vulnerability.
*   We will notify you when the vulnerability is patched.
*   We will credit you for responsible disclosure if you wish to be named publicly (unless you prefer to remain anonymous).

### Scope and Bug Bounty

*   This open-source repository is generally outside the scope of our formal bug bounty program.
*   Reports made through this channel for this repository are not eligible for monetary rewards, but will be handled with priority for remediation.

Thank you for helping keep this project safe!
